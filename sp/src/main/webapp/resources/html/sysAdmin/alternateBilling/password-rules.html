Advanced password security features

Password Characteristics

Maximum length of the password should be 14 characters
Complexity of the password:
No dictionary words included
Password should include the following types of characters Lowercase Uppercase Numbers Special characters such as !@#$%^&*(){}[] Passwords will be case sensitive

Password History

Require a number of unique passwords before an old password may be reused. This number should be no less than 4

Password Age

Maximum password age should be 30 days and minimum should be 2 days.
Password will expire automatically after maximum days and platform will force member to reset the password

Failed login attempts:

Account lockout threshold - 4 failed login attempts
Reset account lockout after - The time it takes between bad login attempts before the count of bad login attempts is cleared. The recommended value as of the date of writing this article is 20 minutes. This means if there are three bad attempts in 20 minutes, the account would be locked.
Account lockout duration would be 2 hours, during this time member will not be able to login but member can reset password through forgot password functionality.
Member can try to login with current password after lockout period.
